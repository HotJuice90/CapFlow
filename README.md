# CapFlow

Персональный центр управления капиталом (React Native + Expo, Android-first).
См. `CapFlow_Specification-5.md` (спека), `CapFlow_Decisions.md` (решения),
`design_guide_for_claude_code.md` (вайб/стек).

## Требования

- **Node.js LTS** (20+). Проверка: `node -v`
- **Android Studio** + Android SDK + эмулятор или устройство с USB-отладкой
- Java JDK (ставится вместе с Android Studio)

## Первый запуск

```bash
npm install
npx expo install --fix      # выравнивает версии нативных модулей под SDK
npx expo-doctor             # проверка окружения (опц.)
npm run android             # сборка dev-client и запуск на эмуляторе/устройстве
```

Если хочется быстрый превью без нативной сборки — `npm start` и открыть в Expo Go
(учитывая, что blur/нативные модули на Go могут отличаться от собранного APK).

## Скрипты

- `npm run android` — запуск на Android
- `npm start` — Metro / Expo dev server
- `npm test` — юнит-тесты (расчётный движок)
- `npm run typecheck` — проверка типов

## Структура

```
app/                     экраны (expo-router)
  (tabs)/                нижняя навигация: главная, календарь, конвертер, аналитика, настройки
  asset/[id].tsx         карточка актива
src/
  calc/                  расчётный движок (чистый TS, под тестами) — единый источник истины
  domain/                продуктовая модель данных
  format/                форматирование денег/процентов/дат (рос. правила)
  storage/               репозиторий (AsyncStorage за абстракцией)
  state/                 контекст данных + селекторы портфеля
  data/seed.ts           демо-портфель (isDemo)
  theme/                 дизайн-токены (светлая тема, токены под тёмную)
  components/            переиспользуемые UI-блоки
  i18n/                  строки (русский, i18n-ready)
```

## Тестовые данные

На первом запуске наполняется демо-портфель. Удалить: **Настройки → Данные →
Удалить тестовые данные** (чистит только демо, помеченные `isDemo`).
