# Дизайн-гайд: светлый «стеклянный» финтех-вайб

Инструкция фиксирует визуальный язык и технический стек, которые мы довели до зрелого состояния в прошлом проекте. Цель — чтобы новый проект придерживался того же вайба: воздушный, светлый, премиальный, не «дефолтный AI-дизайн».

---

## 1. Стек

- **React Native + Expo SDK 56**, RN 0.85.x, React 19.2.x
- **expo-router** — файловая навигация (`app/` папка = экраны, `(tabs)` = группа вкладок)
- **TypeScript** везде
- **react-native-svg** + **react-native-svg-transformer** — для импорта `.svg` как React-компонентов (лого, иконки)
- **@expo/vector-icons** — `MaterialIcons` (filled, основные иконки) + `MaterialCommunityIcons` (outline-вариант для вотермарок/декора)
- **expo-font** — кастомный шрифт, 5 статичных начертаний (Regular/Medium/SemiBold/Bold/ExtraBold), инстансированных из variable-версии (variable fonts капризничают на старом Android)
- **expo-linear-gradient** — фоны, карточки, баннеры
- **@react-native-async-storage/async-storage** — локальное хранилище
- **react-native-reanimated** + **react-native-gesture-handler** — анимации и свайпы
- **expo-haptics** — тактильный отклик на действиях
- **@react-navigation/bottom-tabs** — низовая навигация (с кастомным таб-баром поверх)
- **newArchEnabled: true** — новая архитектура RN включена
- В новом RN (0.85+) тени делаются через **`boxShadow`** (кроссплатформенный CSS-like синтаксис), а не `shadowColor`/`elevation` по отдельности — даёт мягкую тень без жёсткого контура на Android.

Для лого/иконок брендов — **векторные SVG без подложки**, в двух вариантах: цветной (для светлых фонов) и белый-силуэт (для цветных фонов/плашек). Один компонент-обёртка выбирает нужный вариант по контексту.

---

## 2. Общий вайб

Светлый, воздушный, «стеклянный» финтех-стиль. Не тёмная тема, не плоский material-дефолт. Ключевые слова: **мягкость, премиальность, дыхание, точность**.

Что НЕ делаем:
- Эмодзи как иконки (читается как «детское» / дешёвое AI-приложение)
- Жирные чёрные тени с резким контуром
- Линейные градиенты «в лоб» насыщенным цветом (выглядит как заливка, а не дизайн)
- Плоские разнокалиберные иконки из случайных источников — один источник, один стиль
- Перегруженные центрированные карточки — выравнивание чаще влево, воздух важнее симметрии

---

## 3. Фон и поверхности

**Фон экрана** — диагональный линейный градиент из 3 очень светлых пастельных тонов (например мятный → лавандовый → голубой), направление ~168° (не строго вертикально/горизонтально). Локации стопов смещены к краям (0.03 / 0.4 / 0.99), а не равномерно — даёт ощущение естественного света, а не чёткой полосы.

**Белые поверхности (плашки, подложки)** — не плоский белый, а либо:
- чистый `#FFFFFF` с мягкой тенью `boxShadow: 0px 4px 10px rgba(48,69,62,0.10)` (тень тёплая зеленовато-серая, НЕ чёрная),
- либо для «стекла» — полупрозрачный `rgba(255,255,255,0.85-0.97)` с тонкой светлой рамкой `rgba(255,255,255,0.7)` сверху.

**Важно про blur:** `expo-blur` на Android в Expo Go исторически глючил/не рендерился. На SDK 56+ с dev-build/APK нативный blur через RenderNode API работает нормально — но требует обёртки в специальный blur-target компонент и не работает внутри модалок. Если есть сомнение — сначала имитировать стекло через полупрозрачные слои + рамку, blur подключать отдельным шагом на собранном APK, не в Expo Go.

**Подложки могут заезжать друг под друга** — например, нижний блок может начинаться чуть выше верхнего слоя (отрицательный отступ), создавая эффект что один элемент «утоплен» в другой. Это создаёт глубину без теней.

---

## 4. Карточки (ключевой паттерн)

Карточка-акцент (например — карточка категории/сущности с «рейтингом») строится так:

1. **Тень и контент — РАЗНЫЕ слои.** Внешний `View` держит только тень (без `overflow: hidden`), внутренний — `overflow: hidden` для скругления градиента/вотермарки. Если смешать в одном слое — тень либо обрезается, либо вылезает криво.

2. **Градиент — очень светлый пастельный**, не насыщенный цвет. Берётся базовый цвет акцента (категории/бренда/сущности) и подмешивается к белому в пропорции ~88-95% белого. То есть это белый с лёгкой цветной дымкой, а не цветная плашка с прозрачностью сверху — разница принципиальная (если просто наложить rgba(color, 0.15) — будет грязно и ярко).

3. **Иконка-вотермарка** — большая (≈80-100px), полупрозрачная (`opacity: 0.10`), того же цвета что акцент категории, лежит сзади контента, частично срезана краем карточки (`overflow: hidden` обрезает её). Создаёт глубину без необходимости в реальном фото/иллюстрации. Контурный (outline) стиль иконки смотрится мягче, чем filled, для этой роли.

4. **Контент прижат к одному краю** (обычно левому, сверху вниз), не центрирован: подпись → крупное число/процент → ряд мелких бейджей-логотипов внизу.

5. **Бейджи/чипы** (если несколько сущностей привязаны к карточке) — компонуются с перекрытием: первый чип крупнее и выше по z-index, второй — чуть меньше, заходит под первый (`marginLeft: -12` или похоже), создавая ощущение «стека». Если сущностей больше двух — просто символ `+` без числа, без шума.

---

## 5. Списки и строки

Строка списка: иконка-бокс слева (цвет акцента категории, белая иконка внутри, лёгкая светлая рамка `1px #F9FAFB` для отделения от фона) + название (Medium, не Bold) + справа компактные чипы-результаты на нейтральном светлом фоне (`#F9FAFB`, radius 8), где у первого (лучшего/приоритетного) результата текст красится акцентным цветом (не дефолтным зелёным — выбрать свой фирменный акцент, например бирюзовый), у остальных — обычным тёмным.

Если результатов больше двух — отдельный маленький блок `+N` нейтрального серого цвета (`#98A2B7`), не сливающийся с чипами.

Разделитель между строками — тончайшая линия (`1px`, очень светлый серый `#EAEFF8`), не явная graphical border.

Списки почти всегда **сворачиваемые**: показываем 2-3 приоритетных элемента + кнопка «Показать ещё» текстом (не кнопкой с фоном), серым некрикливым цветом.

---

## 6. Горизонтальные слайдеры

Карточки-акценты из п.4 живут в горизонтальном скролле (`ScrollView horizontal`), а не в фиксированной строке — даже если на экране видно только 2-3 штуки целиком + краешек следующей. Это сигнализирует «тут больше, листай» без явной пагинации.

Подписи/баннеры с промо-офером — тоже горизонтальный слайдер: цветной фон (свой на каждую карточку, тёмный насыщенный — это единственное место где можно позволить насыщенный цвет), декоративное светящееся пятно-блик (большой полупрозрачный круг, смещённый за край карточки), и крупная цифра/процент белым в правом нижнем углу.

---

## 7. Таб-бар / нижняя навигация

Плавающий, НЕ прижатый к краям экрана (отступы по 16px слева-справа от края экрана), скруглённый, полупрозрачный или почти непрозрачный белый (`rgba(255,255,255,0.92-0.97)` — на грани, чтобы не было «каши» от просвечивающего контента, но и не совсем плоский).

Если есть центральное CTA-действие (плюс/сканер/главное действие) — оно выходит за пределы бара круглой кнопкой, **утопленной в вырез** самого бара (не просто кнопка поверх плоского бара). Вырез рисуется через SVG-path с дугой, а не имитируется зазором между двумя половинками бара. Кнопка — насыщенный акцентный цвет, с мягким цветным свечением под собой (полупрозрачный круг того же цвета, размытый/большой, позади кнопки) и собственной тенью.

---

## 8. Типографика

- Один шрифт-семья на весь проект, с поддержкой кириллицы если язык — русский (проверять обязательно — много латинских шрифтов кириллицу не тащат).
- Минимум 4-5 начертаний: Regular (текст), Medium (вторичные подписи/названия в строках), SemiBold (заголовки секций), Bold/ExtraBold (крупные числа-акценты — проценты, суммы, метрики).
- Крупные цифры-метрики — всегда Bold, размер заметно больше окружающего текста (28-36px против 13-16px у подписей). Это центр внимания карточки.
- Заголовки экранов — крупные (32-36px), SemiBold, обычный (не letter-spaced) трекинг.

---

## 9. Цвет

- **Один акцентный цвет** на весь интерфейс для «лучшего результата/активного состояния» (в нашем случае — бирюзовый `#18C1AD`). Не путать с цветами категорий/сущностей.
- **У каждой сущности/категории — свой фирменный цвет**, который используется ПОВСЮДУ одинаково: заливка иконки-бокса в списке, цвет вотермарки и градиента в карточке-акценте, акцентная деталь. Один источник цвета на сущность — не выдумывать разные оттенки для разных мест.
- Текст: основной `#212121` (не чистый чёрный), вторичный `#667085`/`#98A2B7` (приглушённый серо-синий, не серый-нейтральный).
- Тени всегда цветные (зеленовато-серые `rgba(48,69,62,0.1)`), никогда чисто чёрные.

## 10. Анимация и интерактив

- Переключение между состояниями (например, между периодами/вкладками контента) — лёгкая анимация смены: старое значение уезжает в сторону + угасает (`opacity`), новое приезжает с противоположной стороны + появляется. Длительность короткая (120-200ms), easing — `ease-in` на выходе, `ease-out cubic` на входе.
- Свайпы должны дублировать видимые кнопки-стрелки, а не быть единственным способом навигации — но если жест есть, дельта для срабатывания небольшая (~40px), направление жеста чётко отличается от вертикального скролла (через сравнение `dx`/`dy` с коэффициентом ~1.5-1.6).
- Пустые состояния (нет данных) — не просто текст «нет данных», а карточка с иконкой, понятным заголовком и подсказкой какое действие приведёт к появлению контента. Пустое состояние должно визуально вписываться в общий вайб (та же скруглённость, тот же мягкий нейтральный фон), а не выбиваться системным видом.

---

## 11. Общее правило при сомнении

Если есть выбор между «выглядит чуть строже/технологичнее» и «выглядит чуть мягче/воздушнее» — выбираем мягче. Если выбор между «насыщенный яркий цвет» и «приглушённый пастельный» — выбираем пастельный, кроме точечных акцентов (промо-баннеры, главная CTA-кнопка), где насыщенный цвет — осознанный контраст на фоне общей мягкости.
